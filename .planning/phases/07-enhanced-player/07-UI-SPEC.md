# Phase 7 — UI Design Contract

> Visual and interaction contract for enhanced player features: skip controls, position memory, and sentence highlighting. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS frontend; @wordpress/components in editor only) |
| Icon library | Inline SVG (extending existing set with skip-back, skip-forward -- 20x20 viewBox, currentColor) |
| Font | `inherit` from WordPress theme (Phase 6 D-03) |

**Continuity:** Phase 7 extends the existing player. Same CSS custom properties with theme.json inheritance (Phase 6), same `data-tts-state` visibility pattern, same vanilla JS approach. No new dependencies.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Progress bar height, gap between label and duration, highlight padding-block |
| sm | 8px | Gap between player elements, resume prompt padding-block |
| md | 16px | Vertical padding inside player container, resume prompt padding-inline |
| lg | 24px | Horizontal padding inside player container |

Exceptions:
- Play button: 48x48px (touch target, carried from Phase 1)
- Skip buttons: 36x36px (secondary action, matches former stop button sizing)
- Speed button: 40x36px minimum (carried from Phase 2)

**Source:** Phase 2 UI-SPEC spacing scale (unchanged). Skip buttons reuse 36px secondary button convention.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--tts-light)` | Player background, resume prompt background |
| Secondary (30%) | `var(--tts-dark)` | Label text, skip button icons, duration text (at 60% opacity), speed button text |
| Accent (10%) | `var(--tts-primary)` | Play button background, progress bar fill, resume action link color, active highlight background |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: play button background, progress bar fill, resume action link text, sentence highlight background (at 15% opacity via `color-mix`).

**New derived property:**
```css
--tts-highlight: color-mix(in srgb, var(--tts-primary) 15%, transparent);
```

This creates a subtle, theme-aware background tint for highlighted sentences. At 15% opacity it is visible during playback without obscuring text readability.

**Source:** Phase 6 theme.json color inheritance. All colors derive from the 3 CSS custom properties already scoped to `.wp-block-tts-js-player`.

## Skip Buttons

| Property | Value |
|----------|-------|
| Size | 36x36px (`min-width: 36px; height: 36px`) |
| Background | `transparent` |
| Border | none |
| Border-radius | 50% (circular, matching play button shape) |
| Icon size | 20x20px SVG, `currentColor` fill |
| Icon color | `var(--tts-dark)` |
| Hover background | `var(--tts-btn-hover)` (same as speed button hover) |
| Active/pressed | `transform: scale(0.95)` (same as play button) |
| Cursor | `pointer` |
| Focus-visible | `outline: 2px solid var(--tts-dark); outline-offset: 2px` (Phase 5 pattern) |

**Layout position (per D-02):**
```
[SKIP-BACK 36x36] [PLAY 48x48] [SKIP-FWD 36x36] [info flex:1] [speed]
```

Skip buttons flank the play button. The three buttons form a media control cluster.

**Visibility (per D-04):** Skip buttons visible only in `playing` and `paused` states. Hidden in `idle`, `loading`, `finished`, `error`. Uses CSS state-driven visibility:
```css
.tts-skip-btn { display: none; }
.wp-block-tts-js-player[data-tts-state="playing"] .tts-skip-btn,
.wp-block-tts-js-player[data-tts-state="paused"] .tts-skip-btn { display: flex; }
```

**SVG icons (per D-05):**

Skip back icon (double left arrow):
```html
<svg class="tts-icon tts-icon--skip-back" width="20" height="20" viewBox="0 0 20 20"
     fill="currentColor" aria-hidden="true">
  <polygon points="10,4 2,10 10,16"/>
  <polygon points="18,4 10,10 18,16"/>
</svg>
```

Skip forward icon (double right arrow):
```html
<svg class="tts-icon tts-icon--skip-forward" width="20" height="20" viewBox="0 0 20 20"
     fill="currentColor" aria-hidden="true">
  <polygon points="2,4 10,10 2,16"/>
  <polygon points="10,4 18,10 10,16"/>
</svg>
```

**Accessibility (per D-16):**
- `aria-label="Vorige zin"` (skip back)
- `aria-label="Volgende zin"` (skip forward)
- Tab order: Skip-back, Play, Skip-forward, Speed button (4 tab stops when visible, 2 when skip buttons hidden)
- `tabindex="-1"` when hidden (idle/finished/error states)

## Resume Prompt

| Property | Value |
|----------|-------|
| Container | Inline element within `.tts-info`, below duration, above progress bar |
| Background | `var(--tts-btn-hover)` (subtle tint, same as button hover) |
| Border-radius | 4px |
| Padding | `8px 16px` |
| Margin | `4px 0 0` (above progress bar area) |
| Max width | 100% of `.tts-info` |

**Content layout:**
```
+-----------------------------------------------------+
| Verder luisteren vanaf zin 12?  [Ga verder] [Opnieuw] |
+-----------------------------------------------------+
```

Single line on desktop. Text + two action links, right-aligned actions.

**Typography:**
- Prompt text: 14px, weight 400, `var(--tts-dark)` at 60% opacity
- Action links: 14px, weight 700, `var(--tts-primary)` color, `cursor: pointer`
- Action link hover: `text-decoration: underline`

**Visibility:**
- Shown only in `idle` state when a saved position exists in localStorage
- Hidden once user clicks either action (resume or start fresh)
- Hidden in all other states (playing, paused, loading, finished, error)

**Mobile (max-width: 480px):**
- Wraps to two lines: prompt text on first line, actions on second line
- Actions left-aligned on mobile (not right-aligned)

**Source:** D-08 (resume UX), D-17 (inline, not modal).

## Sentence Highlighting

| Property | Value |
|----------|-------|
| Injected element | `<span class="tts-hl" data-chunk="N">` wrapping each sentence in article paragraphs |
| Active class | `tts-hl--active` on the currently-spoken chunk |
| Background (active) | `var(--tts-highlight)` = `color-mix(in srgb, var(--tts-primary) 15%, transparent)` |
| Background (inactive) | `transparent` (no visual change for non-active chunks) |
| Padding | `2px 0` (vertical only, prevents text reflow) |
| Border-radius | 2px (subtle rounding on highlight background) |
| Transition | `background-color 0.2s ease` |
| Auto-scroll | `scrollIntoView({ behavior: 'smooth', block: 'center' })` on each chunk transition |

**Reduced motion override:**
```css
@media (prefers-reduced-motion: reduce) {
  .tts-hl { transition: none; }
  /* scrollIntoView behavior set to 'auto' (instant) via JS */
}
```

**DOM lifecycle (per D-11, D-15):**
1. On playback start: `view.js` splits paragraph text nodes using `splitIntoChunks()` output, wraps each in `<span class="tts-hl" data-chunk="N">`
2. On chunk transition: remove `tts-hl--active` from previous chunk, add to current chunk, scroll into view
3. On playback stop/finish: remove all injected `<span>` wrappers, restore original paragraph text nodes

**Editor toggle (per D-14):**
- New block attribute: `enableHighlighting` (boolean, default `true`)
- InspectorControls: `ToggleControl` labeled "Tekst markeren tijdens afspelen" in the existing Settings panel
- When `false`: no spans injected, no highlighting, no auto-scroll
- Data attribute on container: `data-tts-highlight="true|false"` (read by `view.js`)

**Source:** D-11 through D-15, ADV-01 requirement.

## Interaction Contract (Phase 7 Additions)

| Interaction | Behavior |
|-------------|----------|
| Click skip-forward (playing) | Cancel current utterance, increment `currentChunkIndex` (clamped to last), call `playNextChunk()` |
| Click skip-forward (paused) | Increment `currentChunkIndex` (clamped), update progress bar, stay paused (chunk plays on resume) |
| Click skip-forward (at last chunk) | No effect (silent clamp per D-03) |
| Click skip-back (playing) | Cancel current utterance, decrement `currentChunkIndex` (clamped to 0), call `playNextChunk()` |
| Click skip-back (paused) | Decrement `currentChunkIndex` (clamped), update progress bar, stay paused |
| Click skip-back (at first chunk) | No effect (silent clamp per D-03) |
| Click "Ga verder" (resume prompt) | Start playback from saved chunk index via `retryPlayback(savedIndex)`, hide prompt |
| Click "Begin opnieuw" (resume prompt) | Clear saved position from localStorage, start from chunk 0, hide prompt |
| Page load with saved position | Check localStorage for `tts-position-{pathname}`, show resume prompt if found and not expired (7 days) |
| Pause (playing state) | Save `currentChunkIndex` + timestamp to localStorage (per D-07) |
| `beforeunload` event | Save `currentChunkIndex` + timestamp to localStorage (per D-07) |
| Playback reaches end | Clear saved position from localStorage (per D-10) |
| Chunk transition (playing) | Move `tts-hl--active` class to current chunk span, auto-scroll |

**Keyboard additions:**
- Left Arrow on focused skip-back: activate skip-back
- Right Arrow on focused skip-forward: activate skip-forward
- Skip buttons participate in tab order only when visible (playing/paused)

## New DOM Elements (Phase 7)

Added to `render.php` output:

```html
<!-- Skip buttons flanking play button (per D-02) -->
<button type="button" class="tts-skip-btn tts-skip-btn--back"
        aria-label="Vorige zin" tabindex="-1">
  <!-- skip-back SVG -->
</button>

<!-- Existing play button stays here -->

<button type="button" class="tts-skip-btn tts-skip-btn--forward"
        aria-label="Volgende zin" tabindex="-1">
  <!-- skip-forward SVG -->
</button>

<!-- Inside .tts-info, after .tts-duration, before .tts-progress -->
<div class="tts-resume" style="display:none;">
  <span class="tts-resume__text">Verder luisteren vanaf zin 12?</span>
  <button type="button" class="tts-resume__action tts-resume__action--continue">Ga verder</button>
  <button type="button" class="tts-resume__action tts-resume__action--restart">Begin opnieuw</button>
</div>
```

**New data attribute on container:**
```html
data-tts-highlight="true"
```

**Highlight spans (injected by JS at runtime, NOT in render.php):**
```html
<span class="tts-hl" data-chunk="0">First sentence text.</span>
<span class="tts-hl tts-hl--active" data-chunk="1">Currently spoken sentence.</span>
```

# Phase 2 — UI Design Contract

> Visual and interaction contract for the reliable speech engine and full player UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS frontend; @wordpress/components in editor only) |
| Icon library | Inline SVG (extending Phase 1 set with speed icon — custom, no external dependency) |
| Font | Poppins (body/labels), inherited from Chapter42 theme via `font-family: 'Poppins', sans-serif` |

**Continuity:** Phase 2 extends the Phase 1 player. Same CSS custom properties, same `data-tts-state` pattern, same zero-dependency vanilla JS approach.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Progress bar height, gap between label and duration |
| sm | 8px | Gap between speed button text and edges |
| md | 12px | Gap between play button, info section, and controls |
| lg | 16px | Vertical padding inside player container |
| xl | 20px | Horizontal padding inside player container |

Exceptions:
- Play button: 48x48px (touch target, carried from Phase 1)
- Stop button: 36x36px (secondary action, carried from Phase 1)
- Speed button: 40x36px minimum (touch-friendly, text content determines width)
- Progress bar track: 4px height (thin, unobtrusive)

**Source:** Phase 1 UI-SPEC spacing scale (unchanged), Phase 2 CONTEXT.md D-09 (compact speed button).

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Label | 16px | 700 | 1.4 | Poppins |
| Duration / remaining | 14px | 400 | 1.4 | Poppins |
| Speed button | 13px | 600 | 1 | Poppins |

**Rationale:** Phase 2 adds one new text element: the speed multiplier label (e.g. "1x", "1.2x"). At 13px semibold it is compact enough to fit in the button without crowding, distinct from the 14px duration text. Line-height 1 because the speed label is single-line inside a button with vertical centering.

**Source:** Phase 1 UI-SPEC typography (label/duration unchanged), Phase 2 CONTEXT.md D-09 (speed button shows multiplier text only).

## Player Layout — Phase 2 (Desktop)

```
+------------------------------------------------------------------+
|  [PLAY]   Luister naar artikel              [1x]          [STOP] |
|   48x48   ~2 min remaining                  speed          36x36 |
|           [========-----------]  progress bar                    |
+------------------------------------------------------------------+
  ^           ^                      ^           ^             ^
  |           |                      |           |             |
  accent bg   flex:1, column         4px tall    navy text     navy border
  white icon  label + duration       red fill    transparent   hidden idle
              + progress bar         beige track  bg
```

**Container:** Same as Phase 1 — `display: flex; align-items: center; gap: 12px; width: 100%; padding: 16px 20px; background: var(--tts-light); border: 1px solid rgba(0, 45, 73, 0.1); font-family: var(--tts-font); border-radius: 0;`

**Info section changes:** The `.tts-info` flex column now contains three rows:
1. Label text ("Luister naar artikel") — 16px bold
2. Duration text ("~3 min" or "~2 min remaining") — 14px regular
3. Progress bar — 4px tall, full width of info section

**Speed button:** Positioned between info section and stop button.

**Layout order (flex):** `[play-btn] [info (flex:1)] [speed-btn] [stop-btn]`

## Player Layout — Phase 2 (Mobile)

**Breakpoint:** `max-width: 480px`

```
+------------------------------------------+
|  [PLAY]   Luister naar artikel    [STOP] |
|   48x48   ~2 min remaining  [1x]  36x36 |
|           [========-----------]          |
+------------------------------------------+
```

Changes at mobile breakpoint:
- Container padding reduces: `padding: 12px 16px`
- Gap reduces: `gap: 8px`
- Speed button moves inside the info section, inline with the duration row (right-aligned via `display: flex; justify-content: space-between` on the duration row)
- This prevents the player from becoming too wide or wrapping awkwardly

**Breakpoint:** `max-width: 360px` (very small screens)
- Label font-size reduces to 15px
- Duration font-size reduces to 13px
- No other layout changes

**Source:** Phase 2 CONTEXT.md Claude's discretion (mobile breakpoints), PLAY-08 requirement.

## Progress Bar

| Property | Value |
|----------|-------|
| Track height | 4px |
| Track background | `rgba(0, 45, 73, 0.1)` (same as container border) |
| Track border-radius | 2px (exception: rounded ends for thin bar readability) |
| Fill color | `var(--tts-primary)` (`#FF3349`) |
| Fill border-radius | 2px |
| Fill transition | `width 0.3s ease` (smooth per-chunk advance) |
| Container margin | `margin-top: 4px` (spacing above progress bar within info section) |
| Width | 100% of `.tts-info` column |

**Behavior:**
- Progress tracks chunk position: `(completedChunks / totalChunks) * 100%`
- Advances at the end of each chunk (not smoothly within a chunk — speech boundary events are unreliable)
- At idle state: progress bar is hidden (`display: none`)
- At loading state: progress bar visible at 0%
- At playing/paused state: progress bar shows current position
- At finished state: progress bar at 100%, then hides when state resets to idle

**Source:** Phase 2 CONTEXT.md D-05 (chunk-based progress), PLAY-05 requirement.

## Speed Button

| Property | Value |
|----------|-------|
| Min width | 40px |
| Height | 36px |
| Padding | `0 8px` |
| Background | `transparent` |
| Border | `1px solid var(--tts-dark)` |

| Text color | `var(--tts-dark)` |

| Cursor | `pointer` |
| aria-label | `Afspeelsnelheid: {current}x` |

**Cycle values (per D-07):** 0.8x, 0.9x, 1x, 1.1x, 1.2x, 1.3x, 1.4x, 1.5x

**Display format:** Show multiplier with one decimal when not whole (e.g. "0.8x", "1.2x"), no decimal for whole numbers (e.g. "1x").

**Visibility:**
- Always visible (idle through finished). Speed selection persists across plays.
- Disabled during loading state (`pointer-events: none; opacity: 0.5`).

**Source:** Phase 2 CONTEXT.md D-07/D-08/D-09/D-10.

## Player States & Visual Mapping (Phase 2 Update)

The player uses a `data-tts-state` attribute on the container. Phase 2 extends the Phase 1 state table.

| State | Play Btn | Stop Btn | Progress Bar | Duration Text | Speed Btn |
|-------|----------|----------|-------------- |---------------|-----------|
| `idle` | Play icon | Hidden | Hidden | "~N min" | Visible, enabled |
| `loading` | Spinner | Hidden | Visible, 0% | "~N min" | Visible, disabled |
| `playing` | Pause icon | Visible | Shows position | "~N min remaining" | Visible, enabled |
| `paused` | Play icon | Visible | Shows position | "~N min remaining" | Visible, enabled |
| `finished` | Check icon | Hidden | 100%, then hidden | "~N min" (reset) | Visible, enabled |

**State transitions (extending Phase 1):**
- Same as Phase 1, plus:
- Speed button click (any state except loading): cycle to next speed value, update button text, update `aria-label`
- During playing: current chunk finishes at old speed, next chunk uses new speed (per D-10)

**Source:** Phase 1 UI-SPEC states table, Phase 2 CONTEXT.md D-10.

## Duration Display Logic

| Condition | Display |
|-----------|---------|
| Idle / before first play | "~N min" (estimated from word count / 150 wpm, adjusted for current speed) |
| Playing / paused | "~N min remaining" (recalculated based on remaining chunks and current speed) |
| Finished (before reset) | "~N min" (original estimate, static) |
| Speed change while idle | Recalculate and update "~N min" immediately |
| Speed change while playing | Update remaining time at next chunk boundary |

**Minimum display:** "~1 min" (never show "~0 min" — round up).

**Remaining time format:**
- Under 1 minute: "< 1 min remaining"
- 1 minute: "~1 min remaining"
- Multiple minutes: "~N min remaining"

**Source:** Phase 2 CONTEXT.md D-04/D-06, PLAY-04 requirement.

## Voice Error State

When no suitable Dutch voice is found (per D-12):

| Property | Value |
|----------|-------|
| Player state | New state: `error` added to state machine |
| data-tts-state | `"error"` |
| Play button | Disabled, shows exclamation icon or play icon grayed out |
| Label text | "Geen geschikte stem gevonden" (replaces "Luister naar artikel") |
| Duration text | "Probeer een andere browser" (replaces "~N min") |
| Stop button | Hidden |
| Speed button | Hidden |
| Progress bar | Hidden |
| Container opacity | 0.7 (visually muted to indicate non-functional) |

**Error icon (new SVG):**
```
tts-icon--error: circle with exclamation mark, 20x20 viewBox, currentColor
```

**Source:** Phase 2 CONTEXT.md D-12, Phase 3 handles broader error states (ERR-01/02/03) but Phase 2 needs the "no voice" error specifically because voice selection is a Phase 2 feature.

## Interaction Contract (Phase 2 Additions)

| Interaction | Behavior |
|-------------|----------|
| Click speed button (idle/playing/paused) | Cycle to next speed in sequence; update button text; update duration estimate |
| Click speed button (loading) | No effect (button disabled) |
| Click play (idle, first time) | Voice resolution runs; if no voice found, transition to `error` state instead of `loading` |
| Click play (idle, voice cached) | Skip voice resolution, use cached voice; transition to `loading` then `playing` |
| Chunk ends during playback | Advance progress bar; update remaining time; start next chunk |
| All chunks complete | Transition to `finished` state; progress bar at 100% |
| Focus speed button (keyboard) | Browser default focus outline (consistent with Phase 1) |

**Accessibility (Phase 2 additions):**
- Speed button: `aria-label="Afspeelsnelheid: {N}x"` — updated on each cycle
- Progress bar: `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Voortgang"`
- Remaining time: live region `aria-live="polite"` on duration element (announces time changes to screen readers, throttled to once per 30 seconds to avoid noise)

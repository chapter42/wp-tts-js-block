# Phase 1 — UI Design Contract

> Visual and interaction contract for the TTS player block. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS frontend; @wordpress/components in editor only) |
| Icon library | Inline SVG (play, pause, stop, checkmark, spinner — custom, no external dependency) |
| Font | Poppins (body/labels), inherited from Chapter42 theme via `font-family: 'Poppins', sans-serif` |

**Rationale:** WordPress plugin frontend must be zero-dependency. Editor sidebar uses @wordpress/components (provided by WordPress at runtime). Player styling uses CSS custom properties matching Chapter42 brand. No design system framework needed.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between icon and label inline elements |
| sm | 8px | Internal padding for stop button |
| md | 12px | Gap between play button, info section, and stop button |
| lg | 16px | Vertical padding inside player container |
| xl | 20px | Horizontal padding inside player container |

Exceptions:
- Play button: 48x48px (touch target, meets 44px minimum for mobile accessibility)
- Stop button: 36x36px (secondary action, smaller than primary)

**Source:** RESEARCH.md CSS example; D3 (full-width player); mobile touch target standards.

|------|-------|-------|

| Secondary (30%) | `#002D49` (navy) | Label text color, stop button border and icon, duration text (at 60% opacity) |

**Not used in Phase 1:** `#d3fc51` (lime green) and `#8EB5B5` (teal) are Chapter42 brand colors but have no role in the Phase 1 player. Reserved for future phases if needed (e.g., progress bar in Phase 2).

## Player States & Visual Mapping

The player uses a `data-tts-state` attribute on the container. CSS selectors drive all visual changes.

| State | Play Button Shows | Stop Button | Label Text | Duration Text |
|-------|-------------------|-------------|------------|---------------|
| `idle` | Play icon (triangle) | Hidden | "Luister naar artikel" | "~N min" |
| `loading` | Spinner animation | Hidden | "Luister naar artikel" | "~N min" |
| `playing` | Pause icon (two bars) | Visible | "Luister naar artikel" | "~N min" |
| `paused` | Play icon (triangle) | Visible | "Luister naar artikel" | "~N min" |
| `finished` | Checkmark icon | Hidden | "Luister naar artikel" | "~N min" |

**State transitions:**
- `idle` -> click play -> `loading`
- `loading` -> utterance.onstart fires -> `playing`
- `loading` -> 3 second timeout -> `idle` (silent failure for Phase 1; Phase 3 adds error messages)
- `playing` -> click play -> `paused`
- `paused` -> click play -> `playing`
- `playing` or `paused` -> click stop -> `idle`
- `playing` -> utterance.onend fires -> `finished`
- `finished` -> 3 second auto-timeout -> `idle`

**Source:** D8 (loading state), D9 (checkmark then reset), RESEARCH.md Pattern 3 (state machine).

## Player Layout

```
+------------------------------------------------------------------+
|  [PLAY]   Luister naar artikel                          [STOP]   |
|   48x48   ~3 min                                         36x36   |
+------------------------------------------------------------------+
  ^           ^                                              ^
  |           |                                              |
  accent bg   flex:1, column layout                     navy border
  white icon  label bold + duration muted               hidden in idle
```

- Container: `display: flex; align-items: center; gap: 12px; width: 100%; padding: 16px 20px`
- Play button: left-aligned, fixed 48x48px, red background, white icon, no border-radius
- Info section: `flex: 1`, two lines stacked (label on top, duration below)
- Stop button: right-aligned, fixed 36x36px, transparent background, navy border, navy icon
- No border-radius on any element (sharp corners match Chapter42 site aesthetic)

**Source:** D3 (full-width), RESEARCH.md CSS example, Chapter42 brand (border-radius: 0).

## SVG Icons

All icons are inline SVG, 20x20 viewBox, `currentColor` fill for theme adaptability.

| Icon | Element | Description |
|------|---------|-------------|
| Play | `.tts-icon--play` | Right-pointing triangle |
| Pause | `.tts-icon--pause` | Two vertical bars |
| Stop | `.tts-icon--stop` | Filled square |
| Check | `.tts-icon--check` | Checkmark |
| Spinner | `.tts-icon--spinner` | Circular arc, CSS `@keyframes` rotation (1s linear infinite) |

Icons switch visibility via CSS `display: none / inline-block` based on `[data-tts-state]` selectors.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click play (idle) | Start speech synthesis; show loading then playing state |
| Click play (playing) | Pause speech; show paused state |
| Click play (paused) | Resume speech; show playing state |
| Click play (finished) | Restart from beginning |
| Click stop (playing/paused) | Cancel speech; reset to idle state |
| Hover play button | Background changes from `#FF3349` to `#002D49` |
| Hover stop button | Background fills with `#002D49`, icon becomes white |
| Focus (keyboard) | Browser default focus outline (do not override in Phase 1) |

**Accessibility (Phase 1 minimum):**
- Play button: `aria-label` set to the configurable label attribute (default: "Luister naar artikel")
- Stop button: `aria-label="Stop"`
- Both buttons are native `<button>` elements (keyboard accessible by default)
- Focus styles: browser defaults (Phase 2 adds custom focus rings)

## Editor Preview Contract

The Gutenberg editor shows a non-playable visual preview of the player.

| Property | Value |
|----------|-------|
| Appearance | Identical to frontend player layout and styling |
| Interactivity | All buttons `disabled`; no speech synthesis in editor |
| Visual distinction | Class `tts-player--preview` added; slightly reduced opacity (0.8) on buttons to signal non-interactive |
| Sidebar settings | InspectorControls with language dropdown, speed slider, label text field |
| Live update | Changing label in sidebar immediately updates the preview label text |
| Duration | Shows static "~3 min" placeholder (actual duration computed server-side at render time) |

**Source:** D6 (live preview), D7 (sidebar settings), RESEARCH.md Pattern 4.

## CSS Custom Properties

```css
:root {
  --tts-primary: #FF3349;    /* Accent: play button background */
  --tts-dark: #002D49;       /* Navy: text, stop button, hover states */
  --tts-light: #F3EDE2;      /* Beige: player background */
  --tts-font: 'Poppins', sans-serif;
}
```

Plugin defines its own `--tts-*` custom properties rather than referencing theme internals. This avoids coupling to theme CSS variable names and ensures the player works if the theme changes.

**Source:** RESEARCH.md Open Question #1 recommendation.
